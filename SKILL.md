# Swarm Thinking — 群体智能头脑风暴法

### 来源

从开源项目 [MiroFish](https://github.com/666ghj/MiroFish)（盛大集团孵化）的多智能体社会模拟引擎中提炼的思维方法论。

### 核心思想

当面对一个复杂决策/创意问题时，人的思维容易陷入单一视角的盲区。MiroFish 的做法是：**构建多个人格化的利益相关方，让他们在模拟世界里相互碰撞、辩论、博弈，最终从涌现的群体行为中找到最优解。**

本质就是把「不同角色放在同一张桌上对话」，让盲区在碰撞中暴露。

---

## 触发条件

当用户说以下关键词时自动激活：
- 头脑风暴 / 推演 / 发散思维 / 帮我分析 / 评估方案 / 模拟一下
- 和 MiroFish 相关：群体智能 / swarm / 多智能体推演
- 决策类问题：该怎么选 / 怎么办 / 有什么方案

---

## 工作流程（6 步法）

### Step 0 — 需求澄清

先确认用户要推演的是什么：
- 决策问题？→ 推演各方案在多种条件下的结果
- 创意问题？→ 推演不同角度的可行性和盲区
- 预测问题？→ 推演未来态势的多种可能性

确认后输出一行：`🧠 Swarm Thinking 启动 → {问题类型}`

---

### Step 1 — 利益方识别与人格构建

分析问题涉及的所有相关方，为每个构建独立人格卡：

```
【人格卡模板】
🎭 角色：[名称]
📌 立场：[支持/反对/中立/观望]
🎯 核心诉求：[这个角色最在乎什么]
📊 信息水平：[高/中/低] — 这个角色知道多少
😤 情绪基调：[乐观/焦虑/愤怒/冷漠]
⚖️ 影响力权重：[1-5] — 这个角色对结果的影响程度
💭 隐藏假设：[这个角色的盲区或固有偏见]
```

**关键原则**：
- 必须包含对立立场的人格（反对者 + 支持者 + 中立者）
- 至少 4 个人格，但不超过 8 个（太多会发散到无法收敛）
- 信息水平要差异化——有的人知道一些别人不知道的事
- 隐藏假设是关键——找出每个角色的盲区

---

### Step 2 — 多维度发散推演

让每个人格从以下 **4 个维度** 发表观点：

| 维度 | 问题 | 目的 |
|------|------|------|
| 🔭 **时间线** | 短期(1周)、中期(1月)、长期(1年)分别会发生什么？ | 发现滞后效应和长期风险 |
| 💰 **成本/收益** | 对各方来说，代价是什么？收益是什么？ | 发现不对等的利益分配 |
| 🔗 **连锁反应** | 这个决策会触发什么二阶/三阶效应？ | 发现蝴蝶效应 |
| ⚡ **极端场景** | 最乐观结果？最悲观结果？黑天鹅？ | 发现被忽略的风险点 |

**输出格式**（每个维度一个表格）：

```
【🔄 多轮推演 - 第一轮：时间线分析】

| 角色 | 短期(1周内) | 中期(1月内) | 长期(1年) |
|------|-----------|-----------|---------|
| 🎭 角色A | [观点] | [观点] | [观点] |
| 🎭 角色B | [观点] | [观点] | [观点] |
...

🔍 涌现发现：[总结这一轮碰撞中暴露出的关键矛盾/共识]
```

依次完成: 时间线 → 成本收益 → 连锁反应 → 极端场景

---

### Step 3 — 收敛分析

从 4 轮发散推演中提取：

1. **共识区**：所有/多数人格一致认为的点 → 这些基本是对的
2. **分歧区**：人格间对立的点 → 这些需要额外信息或妥协
3. **盲区**：所有人格都没想到/没覆盖的点 → 这是最大的风险
4. **涌现规律**：看似独立的行为在碰撞中形成的模式

---

### Step 4 — 方案归因与排序

基于收敛分析，列出所有可行方案路径，每个方案标注：

```
【方案归因矩阵】

| 方案 | 可行性 | 风险等级 | 共识度 | 盲区覆盖 | 综合评分 |
|------|--------|---------|--------|---------|---------|
| 方案1 | ⭐⭐⭐⭐ | 🟡中 | 高 | 完整 | 8.5/10 |
| 方案2 | ⭐⭐⭐ | 🔴高 | 低 | 有缺口 | 5.0/10 |
...

🏆 推荐方案：[方案名称]
📋 理由：[一句话总结为什么这个方案最优]
⚠️ 关键前提：[这个方案成立的前提条件是什么]
🛡️ 风险对冲：[如果前提不成立，Plan B 是什么]
```

---

### Step 5 — 输出策略简报

最终交付物包含：

```
📊 群体智能分析报告

🎯 问题：[原始问题]
🧩 参与人格：[N个] 个人格完成了 [4] 轮推演
🔑 核心发现：[3条最关键的洞察]
📈 推荐方案：[排名第一的方案 + 理由]
🚨 关键警告：[1-2个最需要警惕的风险]
📎 下一步行动：[可以直接执行的下一步]
```

---

## 注意事项

1. **人格不能是中性的** — 「中立观察者」必须也有隐藏假设和偏见，否则推演无意义
2. **不要跳过极端场景** — 最乐观/最悲观/黑天鹅是暴露盲区的最快方式
3. **发现共识不要急着认同** — 问自己「这个共识有没有可能是集体盲区？」
4. **方案归因必须有数据支撑** — 不能凭感觉排序，必须能追溯「为什么这个方案评分高」
5. **如果一轮推演后所有人格高度一致** — 说明人格设计不够多样化，需要增加对立立场

---

## 设计理念（供子 Agent 参考）

此 Skill 不只是「让人物发表观点」，而是 MiroFish 群体智能的简化实现：

- **人格 = MiroFish 的 Agent Persona**
- **4 维度推演 = MiroFish 的多轮社交模拟**
- **人格式碰撞 = MiroFish 的涌现行为**
- **收敛分析 = MiroFish 的 ReportAgent 合成**
