# Swarm Thinking v3.0 — 群体智能推演引擎

> 来源：[MiroFish](https://github.com/666ghj/MiroFish)（盛大集团孵化）+ [OASIS](https://github.com/camel-ai/oasis)（CAMEL-AI）
> MiroFish Docker: http://192.168.3.180:3300 (后端: 5501)

## 核心思想

当面对复杂决策/创意问题时，人的思维容易陷入单一视角的盲区。MiroFish 的做法是：**构建多个人格化的利益相关方，让他们在模拟世界里相互碰撞、辩论、博弈，最终从涌现的群体行为中找到最优解。**

v3.0 核心升级：**从「一个人装多人」进化到「真正多次元的群体智能」**——用 OpenClaw 的 sessions_spawn 启动独立子 Agent，或用 MiroFish Docker 引擎跑完整的社会模拟。

## 触发条件

当用户说以下关键词时自动激活：

- **头脑风暴类**：头脑风暴 / 推演 / 发散思维 / 模拟一下 / 沙盘
- **分析决策类**：帮我分析 / 评估方案 / 该怎么办 / 有什么方案 / 决策
- **MiroFish 相关**：群体智能 / swarm / 多智能体推演 / 社会模拟
- **预测类**：预测一下 / 会怎样 / 未来走向

激活后输出一行：`🧠 Swarm Thinking v3.0 启动`

## ⚙️ Layer 选择（核心决策）

根据问题复杂度自动选择执行层：

```yaml
路由规则:
  Layer 1 — 轻量模式:
    条件: 利益相关方 ≤4 或 时间跨度 <1月 或 用户说"快速分析"
    方式: 单体 LLM 角色扮演（原 6 步法）
    耗时: 1-2 分钟
    适用: 简单决策、快速头脑风暴

  Layer 2 — 并行 Agent 模式 (默认):
    条件: 利益相关方 5-8 或 时间跨度 1-12月 或 用户说"深度分析"
    方式: sessions_spawn 启动 N 个独立子 Agent 并行推演 + 1-3 轮交叉
    耗时: 3-8 分钟
    适用: 中等复杂度决策、产品方向评估、策略对比

  Layer 3 — MiroFish 引擎模式:
    条件: 利益相关方 8+ 或 时间跨度 >1年 或 需要长期记忆演化 或 用户说"全面模拟/预测"
    方式: 调用 MiroFish Docker API（GraphRAG + 多Agent社会模拟）
    耗时: 5-30 分钟（取决于模拟规模）
    适用: 复杂战略推演、舆情预测、社会动态模拟
```

选择 Layer 后输出：`📡 Layer {N} 模式 — [轻量/并行Agent/MiroFish引擎]`

## 📐 Layer 1：轻量模式（单体 LLM 角色扮演）

> 适用：快速决策、≤4 个利益方。保留原 6 步法作为快速通道。

### Step 0 — 需求澄清

确认推演类型（决策/创意/预测），输出 `🧠 Swarm Thinking 启动 → {问题类型}`

### Step 1 — 人格构建

分析所有利益相关方，构建独立人格卡（4-5 张）：

```
🎭 角色: [名称]
📌 立场: [支持/反对/中立/观望]
🎯 核心诉求: [最在乎什么]
📊 信息水平: [高/中/低]
😤 情绪基调: [乐观/焦虑/愤怒/冷漠]
⚖️ 影响力权重: [1-5]
💭 隐藏假设: [盲区或固有偏见]
```

### Step 2 — 多维度发散

从 **12 维矩阵** 中选 4 个最相关的维度（详见 `dimensions-matrix.md`），每个维度输出推演表格。

默认维度组合：

| 问题类型 | 默认维度 |
|---------|---------|
| 决策 | 时间线 → 成本收益 → 连锁反应 → 极端场景 |
| 创意 | 类比迁移 → 反向思维 → 约束解除 → 极限推演 |
| 预测 | 变量敏感性 → 路径依赖 → 群体动力学 → 黑天鹅猎手 |

### Step 3-5 — 收敛分析 → 方案归因 → 策略简报

与原版相同（保留），参见下方「输出格式」章节。

## 🚀 Layer 2：并行 Agent 模式（核心进化）

> 适用：5-8 个利益方、中等复杂度。用 `sessions_spawn` 启动真正独立的子 Agent。

### 执行协议

```
主 Agent（你）:
  │
  ├─ Phase 1: 人格工厂
  │   ├─ 分析问题 → 生成 5-8 张完整人格卡（见 personas-factory.md）
  │   ├─ 从 12 维矩阵选 3-4 个最相关维度
  │   └─ 输出人格清单给用户确认（可选：说"不用确认"则直接进入 Phase 2）
  │
  ├─ Phase 2: 第一轮并行推演
  │   ├─ 对每个人格 spawn 一个子 Agent（sessions_spawn）
  │   │   ├─ task: 包含人格卡 + 问题 + 当前维度 + 推演要求
  │   │   ├─ context: "isolated"（独立上下文，不互相污染）
  │   │   ├─ agentId: 使用轻量模型（如 DeepSeek V4 Flash）
  │   │   └─ taskName: 格式 "swarm-{角色缩写}-R1"
  │   │
  │   ├─ 用 sessions_yield 等待全部完成
  │   └─ 收集所有 Agent 的首次观点
  │
  ├─ Phase 3: 碰撞交叉（第二轮）
  │   ├─ 整理第一轮每个 Agent 的核心观点（匿名化摘要）
  │   ├─ 重新 spawn 所有 Agent（或向同一 session sessions_send）
  │   │   └─ 附加内容：「以下是其他参与者的首轮观点：[摘要列表]。请回应。」
  │   ├─ sessions_yield 等待
  │   └─ 收集第二轮观点（关注：谁改变了立场？谁被说服了？谁更坚定了？）
  │
  ├─ Phase 4: 第三轮（可选，仅分歧大时）
  │   └─ 同上，但聚焦分歧点
  │
  └─ Phase 5: 收敛合成
      ├─ spawn 一个 ReportAgent（report-agent.md prompt）
      │   └─ 输入：所有轮次的所有观点 + 人格卡清单
      └─ 收到 ReportAgent 报告后 → 提炼输出给用户
```

### 子 Agent 任务模板

当 spawn 每个子 Agent 时，task 内容格式：

```markdown
你是【角色名】。参与群体推演。

## 你的人格
立场: [立场]
核心诉求: [核心诉求]
信息水平: [知道什么 / 不知道什么]
情绪基调: [情绪]
隐藏假设: [偏见/盲区 - 你不会有意识地意识到这些]
利益关系: [与其他人物的关系]
影响力权重: [1-5]

## 推演问题
[问题描述 + 背景]

## 当前推演维度: [维度名]
[维度说明]

## 要求
1. 从你的立场出发，给出经过推理的观点
2. 基于你知道的信息，不要使用你不知道的信息
3. 具体、可验证、有推理链
4. 输出格式：
   【我的判断】... (核心立场 1-2 句)
   【推理链】... (逻辑过程)
   【我看到的盲区】... (你认为是别人忽略的)
   【我的情绪驱动】... (情感因素)

## 第二轮+ 附加说明
以下是第一轮其他参与者的匿名观点摘要：
- 观点A: [摘要]
- 观点B: [摘要]
...
请对你的观点做出修正或坚持（如果坚持，说明为什么别人的观点没有说服你）。
```

### ⚠️ Layer 2 关键规则

1. **每个 Agent 用独立 spawn，context="isolated"** — 不能互相读到对方的上下文
2. **Agent 不知道其他 Agent 的具体身份** — 只看匿名摘要，防预设立场
3. **至少跑 2 轮** — 第一轮给独立视角，第二轮才有碰撞涌现
4. **人格卡必须含隐藏假设** — 不标出来 Agent 自己不会有意识地暴露
5. **Agent 用轻量模型** — 5-8 个 Agent × 2-3 轮，成本可控
6. **ReportAgent 用强模型** — 收敛分析需要更强的推理能力

## 🌐 Layer 3：MiroFish 引擎集成

> 适用：8+ 利益方、长期推演、需要 GraphRAG 和时序记忆。调用 MiroFish Docker 实例。

### 执行协议

```
1. 种子材料准备
   ├─ 搜集相关背景资料（web_search / 文件读取）
   ├─ 整理为结构化的种子文本（事件时间线、关键变量、已知事实）
   └─ 输出给用户确认（"将用以下材料提交 MiroFish 模拟：[摘要]）

2. 提交模拟
   ├─ POST http://192.168.3.180:5501/api/simulation
   │   └─ body: { seed_material, prediction_requirement, agent_count, max_rounds }
   ├─ 获取 simulation_id
   └─ 告知用户："MiroFish 模拟已启动，ID: {id}，预计 {N} 分钟"

3. 监控状态
   ├─ GET http://192.168.3.180:5501/api/simulation/{id}
   └─ 提示用户进度（"模拟进行中 60/100 轮"）

4. 获取报告
   ├─ GET http://192.168.3.180:5501/api/simulation/{id}/report
   └─ 主 Agent 读报告 → 提炼 → 输出策略简报

5. 深度互动（可选）
   ├─ POST http://192.168.3.180:5501/api/simulation/{id}/chat
   │   └─ body: { agent_id, message }
   └─ 可向模拟世界中的任意 Agent 追问
```

> ⚠️ 具体 API 端点见 `mirofish-api.md`。使用 `exec curl` 调用。

## 📊 12 维推演矩阵

从以下 12 维中按问题类型选 3-4 个（完整说明见 `dimensions-matrix.md`）：

### 决策类

| 维度 | 核心问题 | 发现什么 |
|------|---------|---------|
| 🔭 时间线 | 短/中/长期变化 | 滞后效应、长期风险 |
| 💰 成本收益 | 各方净损益 | 利益分配不对等 |
| 🔗 连锁反应 | 1/2/3 阶效应 | 蝴蝶效应、意外后果 |
| ⚡ 极端场景 | 乐观/悲观/黑天鹅 | 尾部风险 |
| ♟️ 博弈均衡 | 纳什均衡在哪 | 策略互动的稳定点 |
| ⚖️ 道德伦理 | 公平性/外部性 | 被伤害的沉默群体 |

### 创意类

| 维度 | 核心问题 | 发现什么 |
|------|---------|---------|
| 🪞 类比迁移 | 其他领域模式 | 跨界创新 |
| 🔄 反向思维 | 目标反转 | 颠覆性视角 |
| ✂️ 约束解除 | 去约束后 | 真正的瓶颈在哪 |
| 🧬 组合创新 | 无关概念融合 | 新品类机会 |
| 🚀 极限推演 | 物理/逻辑极限 | 天花板在哪 |
| 👁️ 用户视角 | 不同用户情感 | 被忽视的需求 |

### 预测类

| 维度 | 核心问题 | 发现什么 |
|------|---------|---------|
| 🎛️ 变量敏感性 | 微调哪个变量剧变 | 关键杠杆点 |
| 🛤️ 路径依赖 | 锁定效应 | 先发优势、沉没成本 |
| 🐑 群体动力学 | 从众/极化 | 沉默螺旋、信息瀑布 |
| 📈 技术 S 曲线 | 当前阶段 | 拐点距离 |
| 🏛️ 制度博弈 | 规则制定者激励 | 制度性约束 |
| 🦅 黑天鹅猎手 | 尾部风险 | 被系统性忽略的灾难 |

## ⚠️ 注意事项

### 通用规则
1. **人格不是中性的** — 每个人都有偏见和盲区
2. **发现共识不急着认同** — 问「这是不是集体盲区？」
3. **极端场景不可跳过** — 最快暴露盲区的方式
4. **如果推演后所有人高度一致** — 人格设计不够多样化

### Layer 2 规则
5. **子 Agent 必须 context="isolated"** — 独立上下文，不互相污染
6. **至少跑 2 轮** — 首轮独立视角 + 次轮碰撞才有涌现
7. **Agent 用轻量模型** — DeepSeek V4 Flash 等，防成本爆炸
8. **ReportAgent 用强模型** — 收敛分析需要 DeepSeek V4 Pro 或 Claude

### Layer 3 规则
9. **MiroFish 模拟前确认种子材料** — 垃圾进垃圾出
10. **长时间模拟告知用户** — 不要让用户干等

## 设计理念

v3.0 是对 MiroFish 群体智能引擎的深度实现：

- **Layer 1** = MiroFish 的简化版（单体 LLM 替代）
- **Layer 2** = OpenClaw 原生并行 Agent（OpenClaw 独有能力 — sessions_spawn）
- **Layer 3** = MiroFish 原生引擎（GraphRAG + OASIS + 时序记忆）
- **人格工厂** = MiroFish 的人设生成模块
- **12 维矩阵** = 从 MiroFish 的 23 种 Agent 动作提炼的推演框架
- **ReportAgent** = MiroFish 的 ReportAgent
- **记忆归档** = MiroFish 的时序记忆系统

每一层都在逼近 MiroFish 的真实能力，但每一层也保留了 OpenClaw 的原生优势。
